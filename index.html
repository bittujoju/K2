Welcome to the K2 Crypto Library project.

Doing cryptography is easy. Doing cryptography right is hard. Mistakes in choice of padding, mode, HAMACs and order of cryptographic primitives often have profound impacts on the underlying security properties one is trying to achieve. The K2 library will make it hard to make mistakes, and easy to inspect the code to see what exactly is being done.

What is the state of the project?

Design documents and the beginnings of various subcomponents of the Java subsystems. See the design docs on the Wiki page. Browse the code using the View Project button at the top of this page.

What should I use instead?

KeyCzar is a very good library that make is easy to do crypto well.

What's wrong with KeyCzar?

See the K2 wiki for the reasons for why we felt we needed to start from scratch and replace KeyCzar with K2.

How can I help?

Request to join the k2-crypt-dev group and we'll gladly find interesting and useful things for you to help out with.
